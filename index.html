1 Introduction

There is a growing recent interest in incorporating learning in reasoning systems. Such efforts fall largely into two different areas that we term Area I and Area II. We give a quick overview of what a reasoning system does before describing these two areas.

In general, a reasoning system can be conceived of as a search through some space. This search usually relies on a number of hand-written heuristics. Theorem provers, which lie at the extreme end of reasoning systems (as even trivial-first order provers are capable of simulating Turing-level computation), make this quite explicit, as you can specify these heuristics as an end-user. For instance, in first-order resolution theorem provers, the goal is to find a sequence of resolution operations given an intial set of clauses C resulting in an empty clause. At any point in the search the prover has to choose a set of clauses from overall set of clauses it has derived. Theorem provers use heuristics such as the size of a clause, the complexity of a clause, age of a clause etc to do so.

Efforts in Area I revolve around selecting or computing an appropriate set of heuristics using some form of learning while not tampering with the rest of the search process. Efforts in Area II aim to learn a function from scratch that does the entire search. While there have been quite significant progress in Area I, there has been very little progress in Area II. We feel that one of the main reasons, is that there is no standard dataset that can leveraged for Area II. Datasets for Area I are built up with learning heuristics as a goal.
